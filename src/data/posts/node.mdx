Es una de las formas más rápidas de desarrollar y ejecutar, código en servidores y de forma muy escalable
<Post image="https://i0.wp.com/c1.staticflickr.com/5/4319/35100293084_1118d36f88.jpg?resize=500%2C404&ssl=1"/>
Este entorno sirve para ejecutar JavaScript fuera del navegador, ya que está orientado a servidor, creado en el 2008

Que sea fuera del navegador nos da muchas ventajas, puesto que podemos prescindir de Chrome, Firefox, Opera, etc. y así poder ejecutarlo en servidores, herramientas, consolas, laptops, IoT, y cualquier dispositivo que pueda correr software.
<Post image="https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcSiWgeoOCoaHU0E00SRuT3PTBcoA6INzoT4nw&usqp=CAU"/>
La ventaja de correr JavaScript en estos entornos es que nos permiten tener servidores que funcionan de forma asíncrona

Este entorno ofrece muchas características como la alta concurrencia, esto es logrado debido a la forma en que JavaScript maneja el asincronismo, de salidas y entradas, logrando así tener muchos procesos al simultáneo en distintos núcleos del procesador, a pesar de que el lenguaje de programación es mono-hilo
<Post image="https://photo-cms-vovworld.zadn.vn/w500/Uploaded/vovworld/vowkuqvp/2018_09_27/tincachmangcongnghiep_XKFO.jpg"/>
Node.js logra hacer todo esto, gracias al MotorV8 liberado por Google en el año 2008, este motor fue creado en C++ y lo que hacer es convertir el código JavaScript en código máquina en lugar de interpretarlo en tiempo real
<Post image="http://softwareengineeringdaily.com/wp-content/uploads/2020/04/v8-google-1-638.jpg"/>
Otra de las características importantes de Node.js son los módulos, estos módulos son todo aquello que no sea sintaxis de programación. Muchos de estos módulos vienen por defecto en el paquete de Node.js. También puedes crear o instalar módulos externos en Node.js, esto se maneja de esta forma para cargar solo los módulos que son esenciales para el proyecto
